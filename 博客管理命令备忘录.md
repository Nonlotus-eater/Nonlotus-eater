# 博客管理命令备忘录

## 🚀 网站部署同步命令

### 1. Git 版本控制

```bash

# 查看当前状态
git status

# 添加所有更改
git add .

# 提交更改
git commit -m "更新博客内容"

# 推送到远程仓库
git push

# 拉取远程更改
git pull
```

### 2. Vercel 部署

```bash
# 如果配置了 Vercel CLI
vercel --prod

# 或者直接推送到 GitHub，Vercel 会自动部署
git push origin main
```

## 💻 跨设备同步代码

### 1. 在新设备上首次克隆

```bash
# 克隆仓库
git clone https://github.com/Nonlotus-eater/Nonlotus-eater.git

# 进入项目目录
cd Nonlotus-eater

# 安装依赖
npm install

# 或者使用 bun
bun install
```

### 2. 日常同步工作流

```bash
# 每次开始工作前，拉取最新代码
git pull origin main

# 安装可能的新依赖
npm install

# 启动开发服务器
npm run dev
```

### 3. 完整的工作流程

```bash
# 1. 拉取最新代码
git pull origin main

# 2. 创建/编辑博客文章
# (在 src/content/blog/ 目录下)

# 3. 本地预览
npm run dev

# 4. 代码质量检查
npm run yijiansilian

# 5. 构建测试
npm run build

# 6. 提交更改
git add .
git commit -m "feat: 添加新博客文章 - [文章标题]"
git push origin main

# 7. Vercel 会自动部署到线上
```

## 📂 重要目录说明

- `src/content/blog/` - 博客文章目录
- `src/assets/` - 图片等静态资源
- `src/site.config.ts` - 网站配置文件
- `src/content.config.ts` - 内容集合配置

## 🔧 常用命令速查

| 命令 | 功能 |
| `npm run dev` | 启动开发服务器 |
| `npm run new` | 创建新博客文章 |
| `npm run yijiansilian` | 一键代码质量检查 |
| `npm run build` | 构建生产版本 |
| `git add . && git commit -m "message" && git push` | 提交并推送更改 |

## 📝 博客文章格式

每篇博客文章需要包含以下 frontmatter：

```yaml
---
title: "文章标题"
description: "文章描述"
publishDate: 2025-01-09
tags: ["标签1", "标签2"]
heroImage:
  src: "/images/cover.jpg"
  alt: "封面图片"
comment: true  # 是否开启评论
---
```

## 🐛 常见问题解决

### 构建失败时

```bash
# 清理缓存重新构建
npm run clean
npm install
npm run build
```

### 类型检查错误时

```bash
# 重新同步类型
npm run sync
npm run check
```

### 依赖问题

```bash
# 重新安装依赖
rm -rf node_modules package-lock.json
npm install
```
